# Contributing

## Development flow
1. `make setup`
2. `make lint`
3. `make test`
4. `make demo`

## Commit style
Use Conventional Commits:
- `feat:` new functionality
- `fix:` bugfix
- `docs:` documentation
- `chore:` maintenance

## Adding features
- Add/update policy docs in `docs/`.
- Add tests in `tests/` or crate unit tests.
- Keep CLI UX consistent with existing flags and error style.
